# JSON 数据处理工具套件

专业的 JSON 格式化与解析平台，提供数据美化、压缩验证、格式转换等功能。所有处理均在浏览器本地完成，确保数据隐私安全。

## 功能特性

### 1. JSON 格式化与压缩 (json-formatter.html)
- **格式化**: 美化 JSON 数据，添加适当的缩进和换行，提高可读性
- **压缩**: 移除所有不必要的空白字符，减小数据大小
- **文件支持**: 支持上传 JSON 文件和下载处理结果
- **一键复制**: 快速复制结果到剪贴板

### 2. JSON 验证 (json-validator.html)
- **语法检查**: 验证 JSON 数据的语法正确性
- **错误定位**: 提供详细的错误信息，包含行号和列号
- **统计信息**: 显示键数量、嵌套层级、字符数等统计数据
- **实时验证**: 可选的实时验证功能，自动检测输入变化

### 3. JSON/XML 转换 (json-xml-converter.html)
- **双向转换**: 支持 JSON 转 XML 和 XML 转 JSON
- **自定义根元素**: 可指定 XML 根元素名称
- **格式保持**: 保持数据的层级结构和值的完整性
- **错误提示**: 明确指出格式验证失败的原因

### 4. JSON/CSV 转换 (json-csv-converter.html)
- **JSON 转 CSV**: 将 JSON 数组转换为 CSV 表格格式
- **CSV 转 JSON**: 将 CSV 数据转换为 JSON 数组
- **嵌套处理**: 自动扁平化嵌套对象或转换为字符串
- **特殊字符**: 正确处理逗号、引号、换行等特殊字符

### 5. JSON 差异比较 (json-diff.html)
- **可视化差异**: 高亮显示两个 JSON 之间的差异
- **颜色标识**: 
  - 绿色 = 新增内容
  - 红色 = 删除内容
  - 黄色 = 修改内容
- **详细对比**: 显示修改前后的值
- **无差异提示**: 当两个 JSON 完全相同时给出明确提示

### 6. JSON Schema 生成与校验 (json-schema.html)
- **自动生成**: 根据 JSON 数据自动生成符合规范的 Schema
- **Schema 验证**: 使用 Schema 验证 JSON 数据是否符合定义
- **详细错误**: 显示不符合 Schema 的字段路径和原因
- **标准支持**: 基于 JSON Schema Draft-07 规范

## 使用方法

### 基本操作
1. 在输入框中粘贴或输入 JSON 数据
2. 点击相应的操作按钮（格式化、压缩、转换等）
3. 查看输出结果
4. 使用复制或下载按钮保存结果

### 文件上传
1. 点击"上传文件"按钮
2. 选择 JSON、XML 或 CSV 文件
3. 文件内容将自动填充到输入区域

### 下载结果
- 下载的文件名自动包含工具名称和时间戳
- 例如：`formatted-2025-12-10T22-15-30.json`

## 转换规则说明

### JSON 转 XML
- 对象键转换为 XML 元素名
- 数组项使用 `<item>` 标签包裹
- 特殊字符自动转义（&, <, >, ", '）
- 添加 XML 声明头

### JSON 转 CSV
- 仅支持对象数组格式
- 第一行为表头（对象的键）
- 嵌套对象自动扁平化（使用点号分隔）
- 数组转换为 JSON 字符串

### CSV 转 JSON
- 第一行作为表头（对象的键）
- 每行数据转换为一个对象
- 正确处理引号内的逗号和换行

## 常见问题

### Q: 为什么我的 JSON 验证失败？
A: 常见原因包括：
- 缺少引号（键和字符串值必须用双引号）
- 多余的逗号（最后一个元素后不能有逗号）
- 使用了单引号（JSON 只支持双引号）
- 注释（JSON 不支持注释）

### Q: JSON 转 CSV 后数据丢失？
A: CSV 格式有限制：
- 只能表示扁平的表格数据
- 嵌套对象会被扁平化或转为字符串
- 建议先检查转换后的结果

### Q: 如何处理大文件？
A: 对于超大文件：
- 考虑分批处理
- 使用压缩功能减小大小
- 升级浏览器以获得更好性能

### Q: 支持离线使用吗？
A: 部分支持：
- 基本功能可离线使用
- JSON Schema 工具需要 CDN 加载 Ajv 库
- 建议在线首次加载后可离线使用
